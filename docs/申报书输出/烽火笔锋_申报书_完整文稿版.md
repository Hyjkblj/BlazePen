## 摘要

《烽火笔锋》项目以1937年7月7日至1945年9月2日抗战历史进程为时间主线，围绕战地记者在高压信息环境中的信息采集、事实核验、报道发布、风险沟通和伦理判断等关键任务，建设一套“历史情境驱动、能力成长导向、证据链可追溯”的智能训练系统。项目面向高校新闻传播、历史教育、思政实践与应急传播教学场景，重点解决传统教学中“实践密度不足、评价偏主观、成长不可连续跟踪”的问题。

项目采用“结构化场景库 + 规则评估 + 大语言模型辅助评估 + 知识追踪 + 自适应推荐 + 复盘报告”技术路线，形成“可训练、可评估、可成长、可复盘”的闭环。系统围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等8项核心能力构建指标体系，输出能力雷达图、成长曲线、风险标签、证据链与改进建议。

截至2026年3月26日，项目已形成可验证工程基础：训练场景库已落地8个结构化场景（6个主线场景+2个分支场景），训练相关API（含媒体任务）共12个，训练域累计自动化测试242项，其中235项通过、1项跳过、6项失败，主链路烟雾测试17项全部通过。结合国家统计局、CNNIC与新华社公开数据，项目具备明确的教育需求规模、技术应用基础与推广环境支撑。

关键词：历史情境训练；战地记者；传播伦理；知识追踪；大语言模型；智能教育

## 一、项目实施的目的、意义

### （一）背景与问题界定

当前高校历史教育与新闻传播教育普遍重视理论讲授和案例分析，但在高压力、信息不完整、风险高的传播情境中，学生往往缺乏足够的决策训练机会。具体表现为：

1. 训练“强度不足”：课堂讨论多、实战回合少，难以形成行为层面的能力迁移。
2. 评价“可解释性不足”：教师主观评价占比较高，缺少结构化证据链与统一评分口径。
3. 反馈“连续性不足”：课程结束后缺少成长轨迹数据，无法准确定位短板并迭代训练路径。

从教学现场看，学生在“知道原则”与“执行原则”之间仍存在明显断层。例如在突发事件场景中，学生能够复述“先核验后发布”原则，但面对“时间压力+多源冲突+情绪化舆情”的组合条件时，往往难以在30-60秒内给出兼顾时效、准确与风险沟通的表达方案。该断层说明，仅凭课堂讲授和一次性作业，难以完成对复杂能力的稳定培养，需要引入高频、可重复、可追踪的训练机制。

### （二）项目建设目的

本项目聚焦“历史真实性约束下的传播能力训练”，直接目标是构建一套可在课程内落地、可在教学评估中复核、可在跨课程推广中复用的智能训练系统，推动学生从“知道原则”走向“在压力中做出可解释决策”。

项目建设坚持三项原则：第一，真实性原则，即训练内容严格锚定历史边界与事实约束，不以戏剧化内容替代关键事实；第二，可解释性原则，即评价结果必须附带证据链、风险标记和改进理由；第三，可迁移性原则，即能力模型与技术架构可用于新闻传播课程、思政实践课程与应急传播训练课程的跨场景复用。

### （三）项目建设意义

1. 教育意义：推动历史教育、传播伦理教育与人工智能教育深度融合，形成“教-练-评-改”一体化实践教学链路。
2. 育人意义：强化学生事实意识、责任意识、风险意识与公共表达能力，提升复杂情境中的社会沟通能力。
3. 技术意义：验证“规则+LLM+KT”在教育训练中的可解释性与可运营性，沉淀可复用技术范式。
4. 推广意义：为新闻传播、思政实践、应急传播与媒介素养课程提供可迁移的训练模板。

进一步看，本项目的价值不仅在于“做出一个系统”，更在于形成一套具有可操作性的教学方法学：教师可据此进行课前能力诊断、课中分层训练、课后复盘改进；学生可基于同一口径持续观察自身成长轨迹，从而实现“目标明确、路径可见、反馈及时、改进可执行”的学习闭环。

### （四）外部数据支撑（立项必要性）

1. 需求规模：国家统计局《2024年国民经济和社会发展统计公报》显示，2024年全国研究生招生135.7万人、在学研究生409.5万人、本专科在校生3891.3万人，说明高等教育实践教学需求基数大。
2. 技术基础：CNNIC第55次《中国互联网络发展状况统计报告》显示，截至2024年12月我国网民规模11.08亿人，生成式人工智能产品用户2.49亿人，AI工具使用具备普及基础。
3. 推广环境：新华社2025年5月报道显示，国家智慧教育公共服务平台注册用户达1.64亿、浏览量超613亿，数字教育平台化推广条件成熟。

上述数据分别对应本项目“有需求、可实施、能推广”三项关键论证：高等教育规模决定了实践教学工具的刚性需求；AI用户基础降低了教学端和学生端的使用门槛；国家级平台的普及程度为项目后续跨校展示、资源共享和成果扩散提供了现实路径。

### （五）项目创新点概述

1. 历史边界与训练任务融合创新：将抗战历史关键节点直接转化为训练任务单元，避免“内容真实但训练无效”或“训练有趣但事实失真”两类常见偏差。
2. 过程证据化评价创新：评价不仅给分，还输出触发规则、风险标签和改进建议，形成可追溯证据链，提升教学评价透明度。
3. 成长轨迹连续化创新：通过回合级状态更新，把离散课堂表现转化为连续成长曲线，为过程性评价提供数据支撑。
4. 推荐闭环可解释创新：下一轮场景推荐明确短板依据和风险修复逻辑，避免“黑箱式推荐”影响教学信任。
5. 工程与教学双尺度协同创新：一方面满足工程可测试、可部署、可维护，另一方面满足教学可组织、可观察、可复盘。

## 二、项目研究现状与分析

### （一）现有教学实践现状

情境模拟与案例训练已成为历史教育和传播教育的重要补充方式，但多数实践仍存在以下共性问题：

1. 情境静态化：案例固定、路径固定，缺少实时反馈驱动的动态分支。
2. 评价单点化：偏重结果评价，忽视决策过程中的证据、风险和纠偏行为。
3. 训练孤岛化：训练记录、评估结果、推荐机制分散，难形成闭环。

除上述问题外，传统训练还普遍存在“成本高、复用低、横向比较难”三类痛点。第一，教师组织一次高质量情境演练往往需要大量备课与现场引导时间；第二，不同教师、不同班级间评价口径差异较大，结果可比性弱；第三，训练结果难沉淀为结构化数据，导致后续课程难以复用前期成果。

### （二）技术发展现状

大语言模型在语义理解、结构化输出、对话生成方面已具备应用能力；知识追踪技术在学习状态建模与个性化推荐方面已被广泛应用。结合规则引擎、场景库和日志审计，可以构建可解释、可追踪、可复盘的智能训练体系。

技术趋势上，教育应用正在从“单轮问答辅助”走向“任务化、过程化、证据化”训练系统。也就是说，系统不再只回答“是什么”，而是逐步支持“在什么条件下如何做、为什么这样做、做得是否合格、下一步如何改进”。这与本项目所强调的回合决策训练、风险控制与复盘改进高度一致。

### （三）本项目切入点与差异化

与通用问答/剧情互动类产品相比，本项目聚焦“历史真实性约束+传播伦理边界+结构化能力评估+连续成长跟踪”，核心差异化体现在：

1. 训练边界清晰：场景均绑定历史节点与任务目标，避免“娱乐化漂移”。
2. 评价证据可查：每轮输出可形成风险标记、评分依据与改进建议。
3. 推荐逻辑可解释：依据短板、风险与阶段特征生成下一轮训练建议。
4. 教学落地导向：强调课程应用、答辩展示与后续推广可复用性。

在教学应用层面，本项目采用“主线场景稳定、分支场景可变”的设计思路：主线用于保证课程目标一致性和历史边界稳定性，分支用于体现学生决策差异和个性化训练路径。该设计既能满足课堂统一组织，又能保留个体成长差异，便于教师开展分层教学与分层评价。

### （四）项目现有基础（工程可行性）

截至2026年3月26日，已形成如下工程基础：

1. 场景资产：8个结构化场景（6主线+2分支）。
2. 能力模型：8项技能维度、6项状态维度。
3. 评估机制：5类关键规则、2类高风险硬惩罚机制。
4. 接口能力：训练相关API（含媒体任务）共12个。
5. 质量基线：训练域测试242项，235项通过、1项跳过、6项失败；主链路烟雾测试17项全部通过。

以上数据说明项目已具备可运行原型与持续迭代条件，能够支撑后续教学验证与成果转化。

### （五）可行性综合判断

从“需求、技术、组织、资源”四个维度评估，本项目具备较强可行性。需求侧有明确课程痛点和高等教育规模支撑；技术侧已有稳定后端主链路与测试基线；组织侧已形成教学与技术协同机制；资源侧已通过复用现有设备与框架控制前期投入。综合判断，项目具备“短周期可交付、中周期可优化、长期可推广”的实施条件。

## 三、项目研究内容和目标

### （一）研究内容

本项目围绕“场景、能力、评估、追踪、推荐、复盘”六个层面开展研究。

第一，研究历史情境驱动的训练场景构建方法。围绕卢沟桥事变、淞沪会战、南京失守、武汉会战、百团大战、终战报道等关键节点，构建可复用、可扩展、可配置的场景库。

第二，研究战地记者能力图谱构建方法。围绕8项核心能力形成结构化指标体系与可量化评分口径。

第三，研究规则评估与LLM评估融合方法。通过规则触发、风险识别、证据提取和语义评估联动，提升评估稳定性与可解释性。

第四，研究知识追踪驱动的状态更新方法。结合BKT/LLM-KT思路，持续记录学习者能力变化轨迹。

第五，研究自适应训练推荐方法。根据能力短板、风险状态与阶段特征进行下一轮场景推荐。

第六，研究训练报告与复盘机制。输出多维报告，服务学生自我改进与教师教学诊断。

六项研究内容将共同支撑三条主线任务：第一条主线是“训练任务标准化”，解决训练对象不一致、口径不统一问题；第二条主线是“评估过程证据化”，解决评价主观化、难复核问题；第三条主线是“能力成长可视化”，解决课程结束后难持续跟踪问题。三条主线并行推进，可确保项目既有研究深度，也有教学落地可操作性。

### （四）关键技术难点与攻关路径

1. 难点一：历史真实性与交互灵活性平衡。攻关路径为“主线历史锚点固定+分支事件可控生成”，避免训练走向失真。
2. 难点二：评估稳定性与语义理解平衡。攻关路径为“规则先行兜底+LLM辅助精评+阈值约束”，降低波动性。
3. 难点三：个性化推荐与教学节奏平衡。攻关路径为“阶段窗口控制+短板优先+风险修复”，确保课堂组织可控。
4. 难点四：工程迭代与教学可用性平衡。攻关路径为“契约测试+烟雾回归+灰度试用”，保证更新不破坏课堂使用。

### （五）预期教学应用场景

2. 实训周场景：作为高密度训练工具，进行多轮挑战与阶段复盘。
3. 竞赛备赛场景：用于答辩展示和能力评测材料生成。
4. 课程考核场景：可与过程性评价结合，形成更客观的能力评价依据。

## 四、项目技术路线（方法）与进度

### （一）总体技术路线

项目采用“历史场景建模 -> 训练交互 -> 联合评估 -> 能力追踪 -> 智能推荐 -> 报告输出”路线。

### （二）关键方法说明

1. 场景建模：以历史时间轴与事件边界约束训练任务，明确背景、目标、风险与行动选项。
2. 训练交互：支持文本/语音交互，记录回合输入、决策路径与过程日志。
3. 联合评估：规则引擎先行识别风险与关键证据，再由LLM辅助语义评估，生成结构化评分。
4. 状态追踪：按回合更新能力状态与风险状态，形成连续成长曲线。
5. 自适应推荐：按短板优先、风险修复、阶段对齐原则推荐下一轮场景。
6. 报告输出：生成能力雷达图、风险分布、错误类型与改进建议，支持教学复盘。

方法执行中将同步记录审计日志，包括回合输入摘要、触发规则、风险标签、推荐理由和报告生成摘要，确保关键训练结论可回溯、可解释、可复核。

## 五、项目预期成果及说明

### （一）预期成果清单

第一，形成《烽火笔锋》训练系统原型平台，具备训练、评估、追踪、推荐、复盘闭环能力。

第二，形成结构化场景库与能力图谱，支持课程实训、教学展示和迭代扩展。

第三，形成“规则评估+LLM辅助评估+知识追踪”融合方法，提升评价连续性与可解释性。

第四，形成多维训练报告模板，包括能力雷达、成长曲线、风险标签、错误分布与改进建议。

第五，形成完整配套文档，包括架构设计、接口文档、部署文档、测试报告、应用说明。

第六，形成可推广教学模式，在新闻传播、历史教育、思政实践与应急传播方向实现迁移应用。

### （二）成果验收建议指标

1. 功能验收：核心训练链路全流程可跑通，关键接口可稳定调用。
2. 质量验收：主链路测试保持100%通过，回归缺陷可追踪闭环。
3. 教学验收：试用后形成可复核的教师反馈与学生成长样例。
4. 文档验收：形成面向后续开发与推广的标准化文档包。

### （三）成果应用与推广路径

1. 校内推广：先在1-2门课程试用，完成教学组织、评价口径与工作流打磨。
2. 校际交流：基于标准化文档和演示环境，在课程共建、教学研讨中进行展示。
3. 场景扩展：在保留核心能力模型前提下，扩展到其他历史阶段或应急传播主题。
4. 持续迭代：以“试用反馈-版本优化-再试用”机制，持续提升训练质量与用户体验。

## 六、项目经费使用情况

### （一）经费使用原则

坚持“服务目标、专款专用、结构合理、结果导向”的原则，优先保障核心能力建设与教学落地环节。

### （二）经费使用方向（建议口径）

一是系统研发与功能完善。用于训练服务优化、前端训练页面完善、报告可视化与接口联调。

二是模型与平台调用。用于大语言模型、语音服务及必要图像服务调用，保障训练交互与评估输出。

三是运行与部署保障。用于本地部署、容器环境、数据库维护、演示环境搭建与稳定性验证。

四是内容建设与样本整理。用于历史资料梳理、场景撰写、规则优化、评分口径细化与样本构建。

五是测试验证与成果展示。用于联调测试、教学试用、答辩演示、材料制作与成果汇报。

### （三）经费执行说明

项目已尽可能复用现有设备与软件环境，前期投入以研发验证和模型服务为主，整体投入节制。后续经费将继续向“核心功能完善、教学验证、成果转化”倾斜，确保项目按节点达成建设目标。

### （四）经费配置建议比例与绩效指标

1. 系统研发与功能完善：建议占比35%-40%，绩效指标为核心功能完成率、缺陷闭环率与版本稳定性。
2. 模型与平台调用：建议占比20%-25%，绩效指标为评估任务完成率、服务可用性与响应时效。
3. 运行与部署保障：建议占比15%-20%，绩效指标为部署成功率、运行稳定性与故障恢复时长。
4. 内容建设与样本整理：建议占比10%-15%，绩效指标为场景扩展数量、样本质量与规则覆盖度。
5. 测试验证与成果展示：建议占比10%-15%，绩效指标为测试通过率、试用反馈覆盖率与答辩材料完整度。

上述比例为建议区间，具体执行可依据校内财务规范与项目阶段重点动态微调，但应坚持“围绕目标、强调产出、可复核评估”的原则。

## 七、实施保障与风险控制

### （一）组织保障

1. 建立“产品/教学/技术”协同机制，确保技术开发与课程需求同步。
2. 建立周节点评审机制，按里程碑检查进度、质量与风险。
3. 建立问题清单与闭环机制，保证需求变更和缺陷修复可追踪。

### （二）风险识别与应对

1. 技术风险：模型输出不稳定。应对策略为规则兜底、阈值控制与证据链审计。
2. 数据风险：历史素材口径不一致。应对策略为来源分级、版本管理与人工复核。
3. 质量风险：接口契约漂移。应对策略为契约测试、回归测试与变更门禁。
4. 教学风险：课程适配不足。应对策略为试用先行、双向反馈与迭代优化。

### （三）合规与伦理边界

1. 坚持历史真实性边界，不以虚构内容替代关键历史事实。
2. 坚持传播伦理底线，避免煽动性表达与风险误导。
3. 坚持教学导向，避免系统娱乐化偏移。

### （四）质量保障与持续改进机制

1. 需求质量保障：关键需求采用“教学方+技术方”双确认，避免后期反复返工。
2. 研发质量保障：核心链路变更必须通过契约测试与烟雾回归，防止稳定性回退。
3. 教学质量保障：每轮试用后形成“问题清单+改进计划+复验结论”三件套。
4. 文档质量保障：版本发布同步更新接口文档、部署文档、测试报告，确保可复用与可交接。
5. 运营质量保障：建立定期复盘机制，按月汇总训练数据、风险分布与优化重点。

## 结语

《烽火笔锋》项目以抗战历史教育为内容底座，以传播决策能力训练为核心目标，以人工智能技术为实现路径，兼顾教育价值、技术深度与实践可落地性。项目坚持历史真实性边界，坚持训练导向与责任导向，具备明确的工程基础和可推广潜力。通过本项目实施，有望形成可复制、可评价、可迭代的高校智能实践教学样板，为历史情境教学和传播能力培养提供高质量解决方案。

从建设逻辑看，本项目不是“单点功能开发”，而是“教学方法、数据标准、技术架构、评估机制”协同推进的系统工程。项目完成后，既可支撑课堂教学改革，也可支撑竞赛展示和校际交流，兼具短期可交付价值与长期可持续价值。基于当前工程基线与明确的阶段目标，项目具备在申报周期内稳定交付、持续优化并形成示范成果的可行性。
