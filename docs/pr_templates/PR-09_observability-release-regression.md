# PR-09 观测性、发布治理与回归体系

## 目标

将项目从“可开发”提升到“可发布、可回归、可定位问题”。

## 依赖

- `PR-06`
- `PR-08`

## 本次前端必须完成

- 增加故事与训练主路径埋点
- 增加错误边界和关键页面兜底
- 建立前端 E2E 冒烟测试基线

## 本次后端必须完成

- 引入结构化日志与 `traceId`
- 为关键接口增加耗时与错误码观测
- 明确数据库迁移规范
- 建立故事与训练主链路集成测试

## 本次不做

- 不做新业务功能
- 不做大规模页面改版
- 不做无关的基础设施替换

## 契约要求

- 所有错误响应带 `traceId`
- 所有关键异步任务可追踪状态
- 发布说明必须列出接口、数据、配置和回滚信息

## 测试要求

### 前端

- 故事主线 E2E 冒烟测试
- 训练主线 E2E 冒烟测试
- 错误边界与失败态测试

### 后端

- 关键接口集成测试
- 日志与 trace 链路验证
- 异步任务状态追踪验证

## Reviewer Focus

### 前端

- 是否覆盖了真正的用户关键路径
- 埋点和错误边界是否只是表面补丁

### 后端

- trace 和日志是否足够定位问题
- 发布和回滚方案是否可执行

## 验收标准

- 故事与训练均有冒烟测试
- 关键错误能关联到 trace
- 发布说明、回滚说明、配置变更清单齐全
