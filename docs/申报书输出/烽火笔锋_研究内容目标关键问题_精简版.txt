（一）研究内容

1. 构建抗战历史情境训练场景库。项目围绕1937年至1945年抗战历史关键节点，结合战地记者在采访、核验、发布、沟通、纠偏与复盘中的真实职责，建设面向传播决策能力培养的历史情境训练场景库。当前已形成8个基础结构化场景，其中包括6个主线场景和2个分支场景；后续将在此基础上持续扩展，逐步形成不少于12个高质量训练场景，服务课堂实训、课后训练和教学展示等应用场景。

2. 建立战地记者传播能力指标体系。项目围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等8项核心能力，构建结构化评价指标体系。评价组织上拟引入 Rubric 评分框架，明确能力维度、等级区间和证据口径，为训练评估、过程诊断和后续强化训练提供统一依据。

3. 研发“规则约束+大模型辅助分析”的联合评估机制。项目拟采用“规则兜底、模型精评、证据留痕、结果可复核”的思路，一方面通过规则体系识别明显失真、违规或高风险行为，另一方面利用大语言模型对学习者决策内容、沟通表达和纠偏行为进行语义分析。为提升评估稳定性，拟结合 Rubric 评分、JSON Schema 输出校验和规则二次校准，推动评价结果更加稳定、可解释、可复核。

4. 建设知识追踪与状态驱动推荐机制。项目拟结合知识追踪与状态建模方法，对学习者在多轮训练中的能力表现、风险暴露和改进轨迹进行持续记录。知识追踪层面，拟以 BKT（Bayesian Knowledge Tracing）作为冷启动阶段的基础方法，并为 LLM-KT、DKT（Deep Knowledge Tracing）等后续扩展预留空间；系统推进层面，则以 `k_state / s_state` 为状态事实源，结合阶段策略和风险标记生成后续训练推荐，实现适度个性化训练。

5. 构建自研 KS 算法体系。项目将在训练主线中凝练自研 KS（Knowledge-State）算法体系，以 `k_state / s_state` 为状态事实源，结合回合评估、风险标记、阶段策略、推荐排序、分支决策、结局判定和复盘诊断共同驱动训练推进。KS 并非单点打分模型，而是一套多策略协同的状态驱动训练引擎，是项目形成差异化训练能力与工程闭环能力的核心支撑。

6. 开发可视化复盘报告系统。项目将围绕训练全过程数据，完善“结局结果+训练报告+诊断视图”三层结果结构，综合展示学习者的关键选择、能力变化、风险表现和改进建议。报告输出将采用能力雷达图、成长曲线、风险分布和行为摘要等方式，服务学生自我复盘和教师过程性指导。

7. 研究国家智慧教育平台适配路径与接口基础。项目将围绕国家智慧教育平台的资源标准、接口规范和结果呈现要求，开展适配研究与接口预留工作。现阶段重点是梳理数据口径、形成对接方案，并结合 OpenAPI 契约描述提升系统接口规范性，为后续接入和推广打下基础。


（二）研究目标

1. 建设一套可用于高校课堂实践教学的抗战战地记者训练系统原型，形成较完整的训练主链路。系统将覆盖训练入口、场景推进、回合评估、状态更新、结果展示和复盘反馈等核心环节，并在现有基础上持续扩展场景资产和优化交互体验。

2. 实现8项核心传播能力的可量化、可评估、可追踪。项目将通过 Rubric 评分框架、LLM 结构化分析、BKT 驱动的状态更新和报告可视化等方式，逐步构建学习者能力成长画像，提升传播实践教学中的过程评价和持续改进能力。

3. 完成系统主链路开发与阶段性性能优化。项目将以主功能可运行、关键链路可验证、核心流程可恢复、主要结果可解释为工程目标，持续优化响应效率、并发承载能力和教学使用稳定性。诸如单轮评估响应时间控制在3秒以内、支持约50人课堂并发等指标，作为阶段性建设目标持续推进。

4. 形成具有研究价值和成果沉淀价值的项目产出。项目将围绕训练场景设计、联合评估机制、BKT/LLM-KT 驱动的知识追踪方法和教学应用验证等内容形成阶段性研究成果，计划产出相关论文与软件著作权，增强项目的规范性、学术性和延展能力。

5. 形成可推广、可复用的实践教学成果体系。除系统原型外，项目还将逐步沉淀场景资源、指标说明、测试报告、教学手册、演示材料和复盘样例，为新闻传播、历史教育、思政实践及相关课程提供可借鉴的数字化训练方案。


（三）拟解决关键问题

1. 历史真实性与训练交互性如何平衡。项目需要在保证抗战历史真实性和价值导向准确的前提下，提升训练的互动性和参与度。为此，将采用“主线史实固定、分支任务可控”的设计思路，在确保史实边界清晰的基础上增强训练有效性。

2. 训练评估如何做到稳定、可解释、可复核。单纯依赖大模型容易带来评价波动，单纯依赖人工又难以规模化。项目将通过规则体系、Rubric 评分、JSON Schema 输出约束和模型辅助分析相结合的方式，提高评估结果的可信度和教学可用性。

3. 个性化推荐如何与课堂节奏协同。项目既要尊重学生能力差异，又要保证课堂组织可控。为此，将结合 BKT 状态更新、`k_state / s_state` 协同建模、阶段窗口控制和风险优先修复机制，在统一教学框架下实现适度差异化训练。

4. 数据安全与教学合规如何保障。项目在运行过程中会涉及学生身份信息、训练记录和评价结果等数据，需要坚持数据最小化、权限分级、过程留痕和可控部署原则，确保系统具备必要的安全基础和合规条件。

5. 工程迭代与教学可用性如何同步保障。项目不仅要实现功能可运行，还要保证接口规范、结构清晰、训练链路可恢复、测试可回归。后续将以 OpenAPI 契约、JSON Schema 结构约束、过程日志和文档沉淀为抓手，推动系统从“可演示”走向“可教学、可维护、可扩展”。


（四）申报书图片素材建议

1. 项目总览图
建议内容：一张总览式结构图，展示“历史情境场景库-联合评估-知识追踪-状态推荐-复盘报告”的整体闭环。
用途：放在项目简介或研究内容开头，帮助评审快速理解项目全貌。

2. 场景库结构图
建议内容：展示“6个主线场景+2个分支场景”的当前基础结构，并用虚线标出“扩展至12个场景”的规划方向。
用途：支撑研究内容中关于场景库建设的描述。

3. 核心能力指标图
建议内容：用八维能力雷达图或能力矩阵图展示8项核心能力及其对应含义。
用途：支撑能力指标体系部分，增强可视化表达。

4. 联合评估机制图
建议内容：画出“规则引擎+Rubric 评分+LLM 语义分析+JSON Schema 校验”的流程图。
用途：支撑评估机制的技术可信度，突出“可解释、可复核”。

5. 知识追踪与推荐机制图
建议内容：展示“学生输入-回合评估-BKT 状态更新-`k_state / s_state` 更新-下一场景推荐”的链路图。
用途：支撑知识追踪、自适应推荐和训练推进机制。

6. 训练界面原型图或系统截图
建议内容：至少准备训练入口页、训练过程页、结果报告页3张图。
用途：体现系统已具备可见的产品形态，增强申报书说服力。

7. 复盘报告示意图
建议内容：包括能力雷达图、成长曲线、风险分布图、行为摘要模块。
用途：支撑“结局结果+训练报告+诊断视图”的成果表达。

8. 技术架构图
建议内容：前端、后端、场景配置、评估引擎、状态存储、报告生成、接口层的分层结构图。
用途：支撑工程可落地性和后续扩展能力。

9. 接口适配示意图
建议内容：展示系统内部数据结构、OpenAPI 接口描述和外部平台适配关系，不需要画得太细。
用途：支撑“国家智慧教育平台适配研究”部分。

10. 测试与验证支撑图
建议内容：测试通过情况截图、主链路 smoke test 结果图、训练流程验证图。
用途：支撑系统稳定性和工程可信度。


（五）图片准备建议

1. 如果申报书偏“教学成果型”，优先准备场景库结构图、能力指标图、训练界面图、复盘报告图。

2. 如果申报书偏“科研技术型”，优先准备联合评估机制图、BKT/状态更新图、技术架构图、接口适配图。

3. 所有图片建议统一风格，优先使用简洁蓝灰配色，避免花哨插画影响正式性。

4. 历史素材图片如需加入，建议只作为背景辅助，优先选择具有公共史料属性的历史照片、战地记者工作场景图、旧报刊版面图，不要让历史图片替代系统结构和技术支撑图。
