# PR-04 故事域服务拆分与异步媒体流水线

## 目标

拆解故事后端巨型服务，分离故事域与媒体域职责，缩短同步热路径。

## 依赖

- `PR-03`

## 本次前端必须完成

- 按领域拆分 API 模块
- 页面只消费结构化资源状态
- 避免在页面层继续猜测资源 URL 是否可用

## 本次后端必须完成

- 从故事服务中拆出会话、回合、结局、媒体资产职责
- 将故事初始化接口归位到 story 域
- 将图片、音频等重任务尽量异步化
- 统一资源状态和资源元数据返回结构

## 本次不做

- 不做前端状态大重构
- 不做故事历史产品页
- 不做训练结果页

## 契约要求

- 资源状态必须显式返回 `pending / ready / failed`
- 资源信息结构化，不再分散在多个隐式 URL 字段

## 测试要求

### 前端

- 资源 ready/pending/failed 三态渲染测试
- 故事主线在媒体失败时仍可继续推进测试

### 后端

- 服务拆分单测
- 资源任务异步化回归测试
- 路由归属回归测试

## Reviewer Focus

### 前端

- 资源状态消费是否统一
- API 模块拆分是否仍有跨域调用

### 后端

- 是否真的完成职责拆分
- 是否仍把媒体逻辑塞在故事热路径

## 验收标准

- 故事服务职责可一句话说明
- 角色域不再承载故事初始化
- 媒体失败不阻塞故事主流程
