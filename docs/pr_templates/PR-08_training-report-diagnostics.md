# PR-08 训练报告与诊断产品化

## 目标

补齐训练进度、报告和诊断展示层，让训练主线具备结果闭环。

## 依赖

- `PR-07`

## 本次前端必须完成

- 新增训练进度页
- 新增训练报告页
- 新增训练诊断页或诊断面板
- 可视化风险标记、能力变化和推荐分支

## 本次后端必须完成

- 提供稳定的训练进度 DTO
- 提供稳定的训练报告 DTO
- 提供稳定的训练诊断 DTO
- 对 recommendation、audit、diagnostics 原始结构做产品化裁剪

## 本次不做

- 不做 BI 级分析平台
- 不做复杂运营后台
- 不做故事主线新增功能

## 契约要求

- 报告 DTO 面向展示
- 诊断 DTO 面向解释
- 不直接把未经整理的内部 payload 暴露给前端页面

## 测试要求

### 前端

- 进度页测试
- 报告页测试
- 诊断页测试
- 空数据与失败状态测试

### 后端

- 进度接口测试
- 报告接口测试
- 诊断接口测试

## Reviewer Focus

### 前端

- 是否存在大量前端二次拼装后端原始结构
- 展示结构是否清晰可读

### 后端

- DTO 是否真正产品化
- 是否暴露过多内部实现细节

## 验收标准

- 完成一次训练后可查看进度、报告和诊断
- 页面无需自行拼复杂内部 payload
- 诊断结果具备可解释性
