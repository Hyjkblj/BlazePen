# PR-03 故事会话持久化与幂等化

## 目标

把故事会话从进程内内存迁移到持久化模型，使其具备恢复能力、幂等能力和审计能力。

## 依赖

- `PR-02`

## 本次前端必须完成

- 调整故事恢复流程，优先依赖服务端会话状态
- 将本地缓存降级为体验层缓存
- 明确展示会话可恢复、需重选、不可恢复三类状态

## 本次后端必须完成

- 建立故事会话持久化结构
- 建立故事回合和快照结构
- 实现故事回合幂等保护
- 提供会话恢复或快照读取能力
- 清理进程内 `_sessions` 作为事实源的职责

## 本次不做

- 不做故事域大拆分
- 不做训练前端接入
- 不做复杂的历史回放产品功能

## 契约要求

- 回合提交响应必须包含当前回合号
- 会话恢复必须返回明确状态
- 快照 DTO 必须稳定且可供前端恢复页面

## 测试要求

### 前端

- 刷新恢复测试
- 会话过期提示测试
- 重复点击或重复提交场景测试

### 后端

- 幂等提交测试
- 服务重启后恢复测试
- 快照读取测试

## Reviewer Focus

### 前端

- 是否仍把 localStorage 当成权威状态
- 恢复路径是否区分清晰

### 后端

- 是否真的移除了内存会话事实源
- 提交幂等和恢复语义是否严谨

## 验收标准

- 后端重启后故事会话可恢复
- 重复提交不会生成错误回合
- 前端刷新后可从服务端恢复状态
