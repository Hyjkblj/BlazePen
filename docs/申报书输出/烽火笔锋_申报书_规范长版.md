## 一、项目实施的目的、意义

本项目围绕“历史情境中的传播决策能力训练”这一核心目标，拟建设一套面向高校新闻传播、历史教育、思政实践及应急传播教学场景的智能化训练系统。项目以1937年7月7日至1945年9月2日抗战历史进程为时间主线，以战地记者在重大历史节点中的信息采集、事实核验、报道发布、风险沟通和伦理判断为主要任务，将人工智能技术融入情境化教学与能力评估过程之中，形成集训练、评估、反馈、复盘于一体的实践平台。

项目实施具有明显的现实必要性。当前相关课程教学大多停留在知识讲授、案例分析与课堂讨论层面，缺少可反复操作、可量化评价、可追踪成长的实战化训练工具，导致学生在面对复杂传播情境时往往“知道原则，却缺乏应用能力”。本项目通过构建历史真实约束下的交互训练环境，使学生在多轮决策中练习史实核验、来源辨析、传播伦理判断、风险表达与公共行动指引能力，从而提升其综合传播素养。

从项目意义看，一是能够推动历史教育、传播伦理教育和人工智能教育的深度融合，探索“教、练、评、改”一体化教学模式；二是能够提升学生在复杂、高压、不完全信息环境中的判断与沟通能力，增强媒介素养与公共责任意识；三是能够形成具有推广价值的数字化训练模板，为高校课程改革和智能教育应用提供新范式；四是能够在技术层面验证大语言模型、知识追踪、结构化场景库和训练推荐机制在教育实践中的应用潜力。

从外部数据看，该项目具有明确的规模化应用基础。国家统计局《2024年国民经济和社会发展统计公报》显示，2024年全国研究生招生135.7万人、在学研究生409.5万人，本专科在校生3891.3万人，说明高等教育实践教学需求基数巨大。CNNIC第55次《中国互联网络发展状况统计报告》显示，截至2024年12月我国网民规模达11.08亿人，其中生成式人工智能产品用户达2.49亿人，AI工具使用已具备较高社会渗透率，为“AI+课程训练”提供了现实技术土壤。

## 二、项目研究现状与分析

从教育实践现状看，历史教育与新闻传播教育虽然普遍重视案例教学，但多数教学环节仍缺乏沉浸式、动态化和个性化的训练工具，难以模拟真实传播环境中的信息不完整、时间压力高、舆情风险大等特点。尤其在高风险沟通、突发事件报道、战时传播伦理等方面，传统教学方式往往依赖教师经验判断，评价结果主观性较强，难以形成标准化、可持续追踪的能力画像。

从技术研究现状看，生成式人工智能、智能对话、知识追踪、向量检索、规则评估与语音交互等技术已具备较好的应用基础，国内外已有一些智能陪练、虚拟仿真、情境教学平台开始尝试引入大语言模型以增强交互能力。然而现有系统大多偏重问答辅助、剧情体验或通用对话，很少同时实现“历史真实性约束、能力结构化评估、训练状态追踪、个性化场景推荐和复盘报告输出”的完整闭环。

结合本项目现有基础分析，项目已由早期的剧情交互系统迭代升级为训练型系统，后端已形成较完整的训练主链路，包括训练初始化、场景推进、回合评估、能力状态更新、训练记录持久化、结局与报告生成等模块，同时已配置场景库、运行参数、推荐策略和训练观测结构。这说明项目不仅具有理论研究基础，而且具备较强的工程落地条件，为后续深化研究和成果推广提供了保障。

从教育数字化发展环境看，项目推广条件持续增强。新华社2025年5月发布的世界数字教育大会信息显示，国家智慧教育公共服务平台注册用户已达1.64亿，浏览量超过613亿，用户覆盖220多个国家和地区，说明“平台化数字教育资源+智能教学应用”的组织形态已形成规模效应。本项目可作为“历史情境+能力训练”细分场景接入该类生态，具备现实迁移基础。

## 三、项目研究内容和目标

本项目的研究内容主要包括以下几个方面：

1. 构建抗战历史训练场景库。围绕卢沟桥事变、淞沪会战、南京失守、武汉会战、百团大战、抗战胜利等关键节点，设计结构化训练任务，明确事件背景、任务目标、风险标签、目标能力和可选行动。
2. 构建战地记者能力图谱。围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等八个维度建立训练评价指标。
3. 研究训练评估机制。采用规则评估与大语言模型语义评估相结合的方法，对学习者每轮输入进行结构化评分，生成证据链和风险标记。
4. 研究知识追踪机制。通过BKT与LLM-KT相结合的方式，动态更新学习者能力状态，实现能力成长过程的量化表达。
5. 研究训练推荐与复盘机制。依据学习者短板、风险状态和历史表现推荐下一轮训练内容，并自动生成可解释的训练报告。

本项目的总体目标是建设一套可运行、可评估、可复盘、可推广的智能训练原型系统。具体目标为：完成训练会话、场景获取、回合提交、能力追踪、结果报告等核心功能开发；形成可用于课程教学和答辩展示的完整训练流程；沉淀一套具备扩展能力的系统架构、数据结构和接口规范；为后续推广应用和深化研究打下基础。

## 四、项目技术路线（方法）与进度

本项目总体采用“历史情境建模—交互训练—多维评估—能力追踪—智能推荐—报告生成”的技术路线。

在技术实现上，首先通过结构化场景库与历史时间轴构建训练任务，并通过历史事实约束机制确保场景边界清晰、内容可靠。其次，依托文本与语音交互方式组织训练过程，使学习者在多轮决策中完成事实判断与沟通表达。再次，系统结合规则评估与大语言模型评估，对学习者输入进行能力映射和风险识别，并将评估结果转换为结构化训练数据。随后，利用知识追踪方法更新能力掌握状态和叙事情境状态，并结合短板补偿、风险补救和阶段对齐等机制生成下一轮场景推荐。最后，将回合过程、评估结果、状态快照、推荐日志、审计事件和复盘结果统一存储，并输出训练报告。

项目实施进度拟分为四个阶段：

1. 需求分析与方案设计阶段。完成能力图谱设计、场景库初稿、技术方案论证和系统架构设计。
2. 核心功能开发阶段。完成训练服务、评估逻辑、状态更新、持久化存储、训练接口及基础前端交互。
3. 优化完善阶段。完成推荐策略优化、报告可视化、场景扩充、测试补强和系统联调。
4. 验证展示阶段。完成教学验证、答辩材料准备、演示流程打磨和成果总结。

结合当前实际开发情况，项目已形成可量化的工程基线：训练域配置了8个结构化场景（6个主线场景+2个分支场景）；能力模型覆盖8项核心技能维度和6项状态维度；规则引擎内置5类关键规则与2类高风险硬惩罚；训练相关API（含媒体任务）共12个；训练域代码规模为32个Python模块、约10272行。测试方面，截至2026年3月26日，训练域自动化测试执行结果为235项通过、6项失败、1项跳过（共242项），其中训练主链路烟雾回归17项全部通过，说明系统已具备持续迭代与教学试用基础。下一步将重点推进前端训练页面、报告可视化、场景内容扩充和教学应用验证，并完成剩余失败用例的契约对齐修复。

## 五、项目预期成果及说明

本项目预期形成以下成果：

1. 形成《烽火笔锋》抗战战地记者决策与沟通能力训练系统原型平台，实现训练、评估、推荐、报告的完整闭环。
2. 形成一套围绕抗战历史关键节点的结构化训练场景库，服务于新闻传播、历史教育与思政课程教学。
3. 形成一套基于规则评估、LLM证据评分与知识追踪融合的训练评价机制，增强评价的客观性、连续性和可解释性。
4. 形成能力雷达图、成长曲线、错误分布、风险标签和复盘建议等多维训练报告，为学生自我提升和教师教学诊断提供依据。
5. 形成系统架构文档、接口文档、数据库设计文档、部署文档及测试文档，为后续二次开发和推广应用提供支撑。

从成果价值看，本项目不仅能够作为课程实践工具使用，还能够作为智能教育、媒介素养训练和历史情境教学的示范型成果进行推广，具有较好的应用前景和扩展空间。

## 六、项目经费使用情况

本项目经费使用坚持“突出重点、统筹安排、节约高效、服务目标”的原则，主要围绕系统研发、模型调用、运行部署、内容建设和成果展示等方面开展。

1. 研发费用。主要用于后端训练链路开发、前端训练与报告页面完善、数据库与接口功能扩展、测试与调优等工作。
2. 模型调用费用。主要用于大语言模型、语音服务及必要的图像服务调用，支撑训练交互、智能评估和结果生成。
3. 运行部署费用。主要用于本地部署环境、Docker联调、数据库运行、服务器测试与系统优化，保障项目演示和使用稳定性。
4. 内容建设费用。主要用于历史资料整理、训练场景撰写、规则词表优化、评分标准细化和测试样本建设。
5. 测试与展示费用。主要用于项目联调测试、教学验证、答辩演示、宣传材料和成果整理。

目前项目以前期研发验证和系统原型建设为主，充分利用现有开发设备、已有软件环境及可复用技术框架，前期投入较为节制，成本控制较为合理。后续经费将继续向核心功能完善、模型服务支撑、报告可视化和教学验证等关键环节倾斜，以确保项目目标顺利实现。
