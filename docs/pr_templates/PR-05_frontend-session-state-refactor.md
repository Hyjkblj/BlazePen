# PR-05 前端会话状态重构与服务端快照接管

## 目标

让前端从流程脚本式状态管理切换为服务端会话快照驱动，收敛状态所有权。

## 依赖

- `PR-03`

## 本次前端必须完成

- 拆分初始化、恢复、缓存、资源恢复逻辑
- 重新定义前端游戏状态层次
- 将 `GameFlowContext` 缩减到入口态和草稿态
- 让游戏页主要消费服务端快照

## 本次后端必须完成

- 提供故事会话读取接口
- 提供故事回合历史读取接口
- 提供快照版本或更新时间信息

## 本次不做

- 不做故事后端服务拆分
- 不做训练前端接入
- 不做新的视觉设计改版

## 契约要求

- 统一故事快照 DTO
- 统一故事历史 DTO
- 明确前端本地兜底字段范围

## 测试要求

### 前端

- 首次初始化测试
- 刷新恢复测试
- 本地缓存损坏恢复测试
- 会话切换测试

### 后端

- 会话读取测试
- 快照返回稳定性测试
- 历史读取测试

## Reviewer Focus

### 前端

- 状态边界是否真正收敛
- 是否减少了本地拼装逻辑

### 后端

- 快照 DTO 是否足够支撑前端恢复
- 历史与快照接口是否稳定

## 验收标准

- 前端不依赖 localStorage 也可恢复基本状态
- 初始化与恢复逻辑可分开测试
- 页面状态职责清晰
