# PR-01 基线治理与命名规范

## 目标

冻结术语、领域边界、接口归属和 review 口径，为后续所有开发建立统一基线。

## 依赖

- 无

## 本次前端必须完成

- 梳理前端状态分类并形成文档
- 明确 `page state / flow state / server session / local cache` 的职责
- 明确故事与训练的路由拆分原则
- 输出前端命名规范，统一 `characterId / threadId / sessionId / sceneId`

## 本次后端必须完成

- 梳理 `character / story / training / media / admin` 的领域边界
- 明确每个现有路由的归属是否合理
- 输出接口清单和术语表
- 标记历史接口中的保留项与废弃项

## 本次不做

- 不做功能开发
- 不改数据库结构
- 不做大规模代码重构
- 不做页面视觉调整

## 契约要求

- 明确统一响应 envelope
- 明确统一错误结构
- 明确故事使用 `threadId`
- 明确训练使用 `sessionId`

## 测试要求

### 前端

- 校对现有状态流转图是否与代码一致
- 校对现有页面路径是否与真实路由一致

### 后端

- 校对现有接口清单是否与路由注册一致
- 校对路由归属是否与领域定义一致

## Reviewer Focus

### 前端

- 状态分类是否足够清楚
- 是否还有混用术语

### 后端

- 域边界定义是否准确
- 是否遗漏已有对外接口

## 验收标准

- 有一份可执行的术语表
- 有一份接口清单
- 有一份状态流转说明
- 后续 PR 可以按同一口径引用本基线
