# PR-02 API 契约统一与错误模型标准化

## 目标

统一故事与训练主线的 DTO、错误码和接口消费方式，消除页面层对后端脏结构和错误文本的依赖。

## 依赖

- `PR-01`

## 本次前端必须完成

- 所有故事和训练接口统一通过 service 层消费
- 所有字段转换收敛到 normalizer
- 移除页面层对错误消息字符串的业务判断
- 统一 API 响应解包逻辑

## 本次后端必须完成

- 统一主要接口的请求和响应 DTO
- 统一错误码模型
- 故事初始化、回合提交、训练初始化、训练提交返回结构对齐
- 对历史接口兼容策略做显式标注

## 本次不做

- 不引入故事会话持久化
- 不做前端状态大重构
- 不做训练新页面开发

## 契约要求

- 统一 `status` 或 `sessionState`
- 统一错误字段 `code / message / details / traceId`
- 明确故事和训练的会话标识字段

## 测试要求

### 前端

- 正常响应解析测试
- 标准错误码分支测试
- 非法字段与缺失字段兼容测试

### 后端

- DTO 单测
- 错误码单测
- 主接口响应结构回归测试

## Reviewer Focus

### 前端

- 页面是否仍残留原始字段判断
- normalizer 是否覆盖完整

### 后端

- 错误码是否可稳定被前端消费
- 同类接口是否仍然返回不同结构

## 验收标准

- 页面不再依赖消息文本判断业务状态
- DTO 文档与实际返回一致
- 新旧接口契约边界明确
