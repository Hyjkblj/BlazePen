（一）研究内容

1. 构建抗战历史情境训练场景库。项目围绕1937年至1945年抗战历史关键节点，结合战地记者采访、核验、发布、沟通与纠偏任务，建设历史情境训练场景库。当前已形成8个基础结构化场景，后续拟扩展至不少于12个高质量场景。

2. 建立传播能力指标体系。项目围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等8项核心能力，构建结构化评价体系，并引入 Rubric 评分框架统一评价口径。

3. 研发联合评估机制。项目拟采用“规则引擎+大模型辅助分析”模式，对学习者决策内容进行结构化评估，并结合 Rubric、JSON Schema 和规则二次校准提升结果稳定性、可解释性和可复核性。

4. 建设知识追踪与推荐机制。项目拟以 BKT 作为冷启动阶段的基础知识追踪方法，并为 LLM-KT、DKT 等后续扩展预留空间；同时结合 `k_state / s_state` 状态建模实现训练推荐与场景推进。

5. 构建自研 KS 算法体系。项目将在训练主线中凝练自研 KS（Knowledge-State）算法体系，以 `k_state / s_state` 为状态事实源，结合回合评估、风险标记、阶段策略、推荐排序、分支决策、结局判定和复盘诊断共同驱动训练推进。KS 并非单点打分模型，而是一套多策略协同的状态驱动训练引擎，是项目形成差异化训练能力与工程闭环能力的核心支撑。

6. 开发复盘报告系统。项目将围绕训练全过程数据，形成“结局结果+训练报告+诊断视图”三层输出，支持能力雷达图、成长曲线、风险分布和改进建议等结果展示。

7. 开展平台适配研究。项目将面向国家智慧教育平台开展接口与资源适配研究，梳理数据口径、预留标准化接口，并结合 OpenAPI 提升系统契约规范性。


（二）研究目标

1. 建设一套可用于高校课堂实践教学的抗战战地记者训练系统原型，形成较完整的训练主链路。

2. 实现8项核心传播能力的可量化、可评估、可追踪，提升传播实践教学中的过程评价能力。

3. 完成系统主链路开发与阶段性性能优化，保障主功能可运行、关键链路可验证、主要结果可解释。

4. 围绕场景设计、联合评估、BKT/LLM-KT 知识追踪和教学应用验证形成阶段性研究成果。

5. 形成可推广、可复用的实践教学成果体系，包括系统原型、场景资源、测试报告、教学手册和演示材料等。


（三）拟解决关键问题

1. 如何在保证抗战历史真实性的前提下提升训练交互性与参与度。

2. 如何通过规则体系、Rubric、JSON Schema 和模型分析提高评估结果的稳定性、可解释性与可复核性。

3. 如何在 BKT 状态更新和 `k_state / s_state` 协同建模基础上，实现个性化推荐与课堂节奏的平衡。

4. 如何在教学试用和后续推广中保障数据安全、隐私保护与应用合规。

5. 如何通过 OpenAPI 契约、结构约束、过程留痕和测试回归，支撑系统持续迭代与教学可用性。
