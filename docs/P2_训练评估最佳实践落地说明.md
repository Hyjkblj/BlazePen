# P2 训练评估最佳实践落地说明

## 1. 目标
- 在不破坏现有训练 API 的前提下，将评估链路升级为：
  - `LLM 结构化评分`
  - `规则校准`
  - `失败自动回退`
- 保证每轮评估都可入库追踪、可审计、可回放。

## 2. 核心实现
- 新增 `backend/training/evaluator.py`
  - `TrainingRoundEvaluator.evaluate_round()` 作为统一入口。
  - 先跑规则基线，再尝试 LLM 结构化评估。
  - 合并逻辑：`llm_weight * llm_delta + rule_weight * rule_delta`。
  - 高风险硬约束（source exposure / unverified publish）强制惩罚。
  - LLM 失败时自动回退到规则结果，并写入 `fallback_reason`。
- 改造 `backend/api/services/training_service.py`
  - 训练回合由 evaluator 产出统一 `evaluation`。
  - 保持原接口字段不变（前端无感升级）。
  - `round_evaluations` 持久化新增 `eval_mode/calibration/fallback_reason`（在 raw_payload 内）。
  - `submit_round` 改为调用原子化持久化入口，减少半成功脏数据风险。
  - 并发重复提交（同 session 同 round）优先幂等返回已落库结果；无法恢复时才返回可解释错误。
  - 场景序列在 `init` 时固化到 `session_meta`，避免发布后代码序列变化影响老会话。

## 3. 环境变量（P2）
- `TRAINING_USE_LLM_EVAL`：是否启用 LLM 评估（默认 `false`）
- `TRAINING_ENFORCE_SCENARIO_ORDER`：是否强制回合提交场景与系统期望场景一致（默认 `true`）
- `TRAINING_LLM_PROVIDER`：`auto/openai/volcengine/dashscope`
- `TRAINING_EVAL_MODEL`：评估模型名（可选）
- `TRAINING_LLM_MAX_TOKENS`：默认 `700`
- `TRAINING_LLM_TEMPERATURE`：默认 `0.2`
- `TRAINING_LLM_MAX_RETRIES`：默认 `2`
- `TRAINING_LLM_RETRY_DELAY`：默认 `0.8`
- `TRAINING_EVAL_LLM_WEIGHT`：默认 `0.70`
- `TRAINING_EVAL_RULE_WEIGHT`：默认 `0.30`
- `TRAINING_MAX_SKILL_DELTA`：默认 `0.20`
- `TRAINING_MAX_S_DELTA`：默认 `0.20`

## 4. 验证
- 新增单测：`backend/test_training_evaluator.py`
- 新增服务编排单测：`backend/test_training_service.py`
- 覆盖场景：
  - 规则模式（LLM 关闭）
  - LLM 成功 + 规则校准
  - LLM 解析失败自动回退
  - 场景顺序校验（默认开启）
  - 场景顺序校验可通过开关关闭
  - 重复回合提交冲突（唯一约束）错误降级
  - 会话级场景序列冻结验证
  - 单轮完结时 ending 持久化验证
  - 并发重复提交的幂等回包验证
