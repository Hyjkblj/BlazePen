# PR-06 故事主线恢复、历史与结局产品完善

## 目标

在架构稳定后补齐故事主线产品能力，完成恢复、历史和结局展示闭环。

## 依赖

- `PR-04`
- `PR-05`

## 本次前端必须完成

- 增加故事恢复入口
- 增加故事历史查看或回放入口
- 增加结局结果呈现页
- 明确会话过期、恢复成功、重选提示的 UI 表达

## 本次后端必须完成

- 提供历史读取接口
- 提供结局结果查询或摘要接口
- 统一结局 DTO 与历史 DTO

## 本次不做

- 不做训练前端接入
- 不做观测性体系全量建设
- 不做新的故事玩法扩展

## 契约要求

- 历史 DTO 至少包含回合号、角色发言、用户选择、状态变化摘要
- 结局 DTO 至少包含结局类型、结局说明、关键状态值

## 测试要求

### 前端

- 恢复入口测试
- 历史查看测试
- 结局展示测试
- 空历史与异常状态测试

### 后端

- 历史接口测试
- 结局接口测试
- 历史回放不影响主会话状态测试

## Reviewer Focus

### 前端

- 产品流是否闭环
- 是否误把回放页与主会话状态耦合

### 后端

- 历史和结局 DTO 是否面向产品而不是面向内部结构
- 查询接口是否会污染主会话

## 验收标准

- 用户可以恢复最近故事会话
- 用户可以查看历史
- 用户可以明确看到结局结果
