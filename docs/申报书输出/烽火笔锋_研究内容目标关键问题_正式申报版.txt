（一）研究内容

1. 构建抗战历史情境训练场景库，夯实项目内容底座与教学载体。本项目拟围绕1937年至1945年中国抗日战争历史进程中的关键节点，聚焦卢沟桥事变、淞沪会战、平型关战斗、百团大战等重要历史背景，结合战地记者在复杂传播环境中的采访、核验、发布、沟通、纠偏与复盘职责，建设面向传播决策能力培养的历史情境训练场景库。项目坚持“主线史实有据、分支任务可控、训练目标明确、教学使用可落地”的建设原则，在历史教师审核与史料校核基础上，对场景的时间背景、地点环境、任务目标、角色约束、风险条件和行动选项进行结构化设计。结合当前工程进展，项目已形成8个基础结构化场景，其中包括6个主线场景和2个分支场景；后续将在此基础上持续扩展和打磨，逐步形成不少于12个高质量训练场景，以满足课堂实训、课后训练、教学演示和成果展示等多种应用需求。

2. 建立战地记者传播能力指标体系，形成可量化、可解释的训练评价基础。项目将结合新闻传播实践教学需求、历史情境传播任务特征以及高校实践育人要求，构建面向战地记者角色训练的核心能力指标体系。现阶段重点围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等8项核心能力展开设计，并进一步明确各能力维度在不同场景中的表现要求、评价证据和等级划分标准。在评价组织上，项目拟引入 Rubric 评分框架，对各能力项形成较为稳定的维度描述、等级区间和证据口径。通过构建结构化指标、Rubric 评分口径与证据链要求相结合的评价框架，使系统不仅能够对学习者的训练表现进行量化判断，还能够解释评价依据、识别薄弱环节并支撑后续针对性训练，从而增强评价结果的稳定性、客观性与教学可用性。

3. 研发“规则约束+大模型辅助分析”的联合评估机制，提升训练评价的稳定性与可解释性。针对教育场景中“纯模型评估波动较大、纯人工评估难以规模化”的现实问题，项目拟构建由规则体系和大模型分析协同组成的训练评估机制。一方面，围绕历史真实性边界、传播伦理要求、风险表达规范、关键信息核验要求等内容建立基础规则体系，并结合 Rubric 评分引擎对明显违规、失真或高风险行为进行稳定识别与约束；另一方面，引入大语言模型对学习者的决策内容、表达方式、核验过程、纠偏行为和沟通策略进行语义分析，提取证据要点、识别风险标签并生成解释性评价结果。为提升输出稳定性，项目还将采用结构化提示约束、JSON Schema 输出校验和规则二次校准等方式，推动 LLM 评估结果从“可生成”进一步走向“可约束、可解析、可复核”。通过“规则兜底、模型精评、证据留痕、结果可复核”的方式，逐步形成兼顾准确性、稳定性与教学解释力的评估模式，为课堂应用与后续研究提供可靠支撑。

4. 建设知识追踪与状态驱动推荐机制，增强训练过程的连续性与个性化。项目将结合知识追踪思路、训练状态建模和阶段性教学组织要求，对学习者在多轮训练中的能力表现、风险暴露、状态变化和改进轨迹进行持续记录与分析。在知识追踪层面，项目拟以 BKT（Bayesian Knowledge Tracing）作为冷启动阶段的基础方法，用于对核心能力项掌握程度进行初步更新；在后续样本积累基础上，为 LLM-KT 和 DKT（Deep Knowledge Tracing）等扩展方案预留研究与迭代空间。当前系统训练主链路已围绕 `k_state / s_state` 状态事实源、风险标记、阶段策略、推荐结果和分支推进机制形成较为清晰的状态驱动框架，后续研究将在此基础上进一步完善状态建模与训练推荐逻辑。具体而言，系统将根据学习者在不同轮次中暴露出的能力短板、风险类型、行为模式和阶段表现，动态生成后续训练场景、复训任务和强化建议，实现从统一训练向差异化训练的有序过渡。该机制既要服务个体能力提升，也要兼顾课堂教学的组织节奏和实施约束，确保推荐逻辑具备教学合理性、工程可实现性和结果可解释性。

5. 构建自研 KS 算法体系，强化训练推进与结果生成的核心方法支撑。项目将在现有训练主链路基础上进一步凝练自研 KS（Knowledge-State）算法体系，以 `k_state / s_state` 作为训练状态事实源，结合回合评估、风险标记、阶段策略、推荐排序、分支决策、结局判定和复盘诊断等关键环节，形成多策略协同的状态驱动训练引擎。KS 并非单纯的打分模型或单轮决策函数，而是面向训练全过程的状态更新与路径推进体系，能够将学习者输入转化为能力变化、情境变化和后续训练决策，从而支撑训练过程的可运行、可恢复、可解释与可复盘。该算法体系既是项目在工程层面形成训练闭环的核心支撑，也是后续沉淀方法论、形成特色研究成果的重要基础。

6. 开发可视化复盘报告系统，形成结果呈现、过程诊断与改进建议的完整闭环。项目拟围绕训练全过程数据建设可视化复盘报告系统，对学习者在训练中的关键选择、能力变化、风险表现、证据链条和阶段结果进行系统汇总与展示。结合当前系统输出结构，项目将继续完善“结局结果+训练报告+诊断视图”三层结果模型，使之既能服务学生个人复盘，也能服务教师课堂评价和教学诊断。报告系统计划采用能力雷达图、成长曲线、风险分布、错误类型统计、关键行为摘要和改进建议等多种方式呈现训练结果，帮助学习者理解自身在整条训练链路中的决策依据、优势能力、短板能力和风险暴露点，同时帮助教师掌握学生的整体训练情况与阶段性共性问题，提高实践教学中的反馈效率与指导精度。

7. 研究国家智慧教育平台适配路径与接口基础，提升项目后续推广与资源共享能力。项目将依据国家智慧教育平台在资源组织、数据结构、接口规范、结果呈现和接入方式等方面的要求，开展适配研究与技术准备工作。现阶段目标不是直接宣称已完成平台接入，而是在项目实施过程中同步梳理资源标准、整理数据输出口径、预留标准化接口、形成对接方案与说明文档，并结合 OpenAPI 接口描述与结构化数据约定逐步提升系统对外契约的规范性，为后续接入、共享与推广奠定基础。通过前期适配研究，项目有望逐步形成更加规范的资源表达与系统接口能力，增强成果转化潜力和跨平台应用能力，服务后续更大范围的课程试用与教学推广。


（二）研究目标

1. 建设一套面向高校课堂实践教学的抗战战地记者训练系统原型，形成较完整的训练主链路。项目总体目标是围绕历史情境训练、能力评价、状态追踪、场景推荐和复盘反馈等关键环节，建设一套具有教学应用价值和持续迭代能力的智能训练系统。系统将覆盖训练入口、角色设定、场景推进、回合评估、状态更新、结果展示和复盘报告等核心功能，逐步打通从“进入训练”到“完成复盘”的完整流程。在现有8个基础场景和训练主链路基础上，项目将继续扩展场景资产、优化交互流程、完善报告输出，使系统能够更稳定地支撑课堂实训、课后练习和项目展示等场景。

2. 实现8项核心传播能力的可量化、可评估、可追踪，逐步完善实践教学评价支撑体系。项目拟围绕史实核验、来源可信评估、时效与准确平衡、风险沟通表达、伦理与安全边界、反谣与纠偏、公共行动指引、沟通闭环管理等8项核心能力，建立相对稳定的评价口径、证据提取方式和结果表达机制。通过 Rubric 评分框架、规则判断、LLM 结构化分析、BKT 驱动的状态更新以及报告可视化等手段，逐步构建学习者能力成长的连续画像，使教师能够从单次表现评价延伸到过程性、阶段性和发展性评价，推动传播实践教学从“结果判定”走向“过程诊断+持续改进”的评价模式。

3. 完成系统主链路开发与阶段性性能优化，形成可用于教学试用的工程基础。项目将以“主功能可运行、关键链路可验证、核心流程可恢复、主要结果可解释”为工程建设目标，持续推进系统稳定性和教学可用性优化。在性能方面，项目将把单轮评估响应效率、课堂并发承载能力、异常恢复能力和训练过程稳定性作为阶段性技术指标进行验证和优化；在教学使用方面，将重点保障训练流程顺畅、反馈信息清晰、报告结构稳定、核心路径测试通过，为后续教学试用和成果展示提供可靠支撑。对于如单轮评估响应时间控制在3秒以内、支持约50人课堂并发使用等指标，项目将其作为阶段性建设目标和验收目标持续推进，而不将其表述为既成事实。

4. 形成兼具研究价值与成果沉淀价值的项目产出，提升项目的学术性与规范性。项目除完成系统建设外，还将围绕训练场景设计、联合评估机制、BKT/LLM-KT 驱动的知识追踪方法、状态驱动训练以及教学应用验证等方面形成阶段性研究成果。项目计划结合实施过程中的技术方案、教学试用结果和系统优化经验，形成1篇相关研究论文，并申请1项计算机软件著作权，推动项目从教学工具研发进一步走向方法研究、成果规范化和知识产权沉淀。通过系统文档、接口文档、测试材料、教学说明和研究文本等成果积累，提升项目的可复用性、可推广性和后续延展能力。

5. 形成可推广、可复用、可持续演进的实践教学成果体系。项目将不仅关注系统本身的开发完成度，也重视围绕系统形成一套可用于课程实践和成果转化的配套资源。具体包括训练系统原型、场景资源、指标口径说明、测试报告、教学手册、演示材料和复盘样例等。通过逐步完善这些配套成果，项目有望为新闻传播、历史教育、思政实践、媒介素养教育以及相关实践课程提供可借鉴的数字化训练方案，并为后续横向扩展至应急传播训练、公共沟通训练等方向保留方法基础和工程接口。


（三）拟解决关键问题

1. 历史真实性约束与训练交互灵活性之间的平衡问题。抗战历史题材具有明确的史实边界、价值导向和教育属性，训练系统必须在保证历史真实性和立场准确性的前提下提升交互性与参与度。如果只强调历史还原，系统容易停留在静态知识展示层面，难以形成训练效果；如果过度追求互动自由，则容易导致场景失真、价值偏移甚至削弱教学严肃性。为解决这一问题，项目拟采用“主线史实固定、分支任务可控、任务目标清晰、风险边界明确”的建设思路，在重要历史节点上坚持史实锚定，在互动层面通过受控分支和训练任务设计增强参与感，从而实现历史严谨性与训练有效性的统一。

2. 训练评估结果的稳定性、可解释性与可复核性问题。在教育场景中，评价结果如果波动较大、依据不清或无法复核，将直接影响教师使用信心和教学应用效果。单纯依赖大模型容易出现判断不稳定、标准漂移和解释不足的问题，单纯依赖人工评价又存在成本高、效率低、口径不统一等现实限制。为此，项目将坚持“规则先行、模型辅助、证据留痕、结果可追溯”的思路，构建稳定的规则基础、清晰的评价口径和结构化证据表达机制，并通过 Rubric 评分框架、JSON Schema 输出约束、规则二次校准等方式降低 LLM 评估漂移风险，使每一次评价不仅给出判断结果，还能够说明其依据、风险点和改进方向，从而提升训练评估在教学场景中的可信度和可用性。

3. 个性化训练推荐与课堂教学节奏协同之间的协调问题。实践教学既要求尊重学生能力差异，又要求保持课堂组织的整体节奏和教学目标的一致性。如果推荐机制过于个体化，可能导致训练路径过于分散、教师难以组织和比较；如果完全采用统一训练路径，则难以针对不同学生的能力短板进行有效强化。项目拟通过阶段窗口控制、场景分层组织、BKT 驱动的能力状态更新、`k_state / s_state` 协同建模以及风险优先修复机制，在保证课堂整体进度可控的前提下，为学生提供适度差异化的训练路径和强化任务，努力实现“同一教学框架下的有限个性化”，提高系统在真实课程环境中的落地能力。

4. 数据安全、隐私保护与教学应用合规问题。训练系统在运行过程中会涉及学生身份信息、训练过程记录、评价结果、能力画像和报告输出等多类数据，必须严格遵循个人信息保护、教育数据管理和安全使用等相关规范要求。如果数据采集边界不清、权限控制不足或部署方式不合理，可能带来隐私泄露、数据滥用和教学风险。为此，项目将坚持数据最小化、权限分级、过程留痕、可控部署和结果审计等原则，在系统设计与应用实施中同步考虑数据安全与合规要求，确保项目在教学试用、后续推广和平台适配过程中具备必要的安全基础和规范条件。

5. 工程可持续迭代与教学可用性同步保障的问题。教育类训练系统不同于单次演示原型，其价值不仅取决于是否能够跑通功能，还取决于是否具备稳定的契约边界、清晰的数据结构、可恢复的训练链路、可维护的场景配置方式和可持续迭代的测试基础。如果在建设过程中只追求短期功能堆叠，而忽视系统结构清晰性与测试保护，后续一旦进入教学试用阶段，便容易出现结果不稳定、改动成本高、问题难定位等情况。为解决这一问题，项目将把主链路稳定、OpenAPI 契约规范、JSON Schema 结构约束、过程留痕、测试回归和文档沉淀作为工程建设的重要组成部分，推动系统在“可演示”基础上逐步迈向“可教学、可维护、可扩展”的成熟形态。
