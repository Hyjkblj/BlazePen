# 烽火笔锋：NotebookLM 一键生成PPT素材（可直接上传）

## 使用说明（给操作者）
- 本文档已按“PPT页”组织。
- 每页包含：`页面标题`、`页面要点`、`讲稿提示`、`可视化建议`。
- 适合 8-12 分钟路演；可裁剪为 6 页精简版。

---

## 第1页 项目定位
### 页面标题
烽火笔锋：历史情境驱动的智能训练系统

### 页面要点
- 面向高校新闻传播、历史教育、思政实践、应急传播课程。
- 以 1937-1945 历史节点为训练主线，强调史实边界与责任导向。
- 目标不是“剧情通关”，而是“能力成长”：可训练、可评估、可复盘。

### 讲稿提示
我们做的不是一个会聊天的剧情工具，而是一套会根据学生表现动态调整训练路径的系统。

### 可视化建议
- 一张“训练闭环图”：训练输入 -> 过程评估 -> 状态更新 -> 个性化推荐 -> 报告复盘。

## 第2页 现实痛点
### 页面标题
为什么传统教学很难完成这类能力培养

### 页面要点
- 痛点1：课堂重知识讲授，缺少高压情境实训。
- 痛点2：评价偏主观，难以形成连续能力画像。
- 痛点3：缺乏可追踪复盘，改进建议不落地。
- 结果：学生“知道原则”，但在时间压力+多源冲突+舆情波动下难以做出稳定决策。

### 讲稿提示
问题不在学生不知道规则，而在真实压力条件下无法稳定执行规则。

### 可视化建议
- 三栏痛点图：场景不足 / 评价粗放 / 闭环缺失。

## 第4页 总体技术架构
### 页面标题
技术架构：KS状态内核 + 多Agent协同集群 + 异步媒体管线

### 页面要点
- 底座：KS状态驱动内核（k_state/s_state/risk_flags/phase）。
- 中层：多Agent协同（导演、推荐、剧本、行为画像）。
- 外层：训练服务编排、观测审计、报告诊断。
- 媒体层：生图/音频异步任务，不阻塞训练主链路。

### 讲稿提示
我们不是单Agent“全都做”，而是多Agent分工协作：每个Agent只做自己最擅长的一件事。

### 可视化建议
- 分层架构图（4层）：前端交互层/训练编排层/Agent协同层/数据与媒体层。

## 第5页 核心算法设计（大白话）
### 页面标题
核心算法：系统像“会调整路线的训练教练”

### 页面要点
- 每轮输入：用户选择 + 场景上下文 + 历史状态。
- 系统先判断两件事：
- 第一，你的能力是涨了还是掉了（k_state）。
- 第二，当前局面是更稳了还是更危险了（s_state）。
- 系统再做三件事：
- 给出本轮评估，更新状态。
- 推荐下一轮更合适场景（训练而非随机剧情）。
- 留下证据链，供最终报告复盘。

### 讲稿提示
我们核心不是“这题对错”，而是“你现在处于什么状态，下一步该练什么”。

### 可视化建议
- 一条流程箭头：用户决策 -> 评估 -> 状态更新 -> 推荐/分支 -> 报告。

## 第6页 导演Agent设计
### 页面标题
导演Agent：负责“控节奏”和“保安全”

### 页面要点
- 角色定位：总导演+调度员，不写台词。
- 输入：当前轮次、k_state、s_state、最近风险轨迹、运行时标记。
- 输出：ExecutionPlan（执行计划）。
- 关键动作：
- 连续高风险 -> 强制低风险场景。
- 能力明显偏弱 -> 触发剧本刷新。
- 局势过于混乱 -> 提高评估重试预算。

### 讲稿提示
导演Agent的价值是“控场”，确保训练不会越练越乱。

### 可视化建议
- 决策树：风险升高/能力偏弱/局势混乱 -> 对应调度动作。

## 第7页 剧本Agent设计
### 页面标题
剧本Agent：负责“内容生成”和“连续性”

### 页面要点
- 角色定位：编剧，不决定训练结构。
- 输入：已冻结场景序列（主场景+小场景）+ 玩家画像。
- 输出：每个 scenario_id 对应叙事内容：
- monologue（独白）
- dialogue（对话）
- bridge_summary（承上启下）
- options_narrative（选项表达）
- 约束：不改场景ID、不改主线顺序、只做内容填充。

### 讲稿提示
导演管“往哪走”，剧本管“这一轮怎么演”，分工清楚才能稳定。

### 可视化建议
- 输入输出框图：冻结结构输入 -> 叙事内容输出。

## 第8页 双Agent协同机制
### 页面标题
导演Agent + 剧本Agent：一个定方向，一个写内容

### 页面要点
- 用户提交后，核心状态先更新。
- 导演Agent基于新状态给出下一轮执行计划。
- 剧本Agent按计划生成下一场景叙事。
- 系统持续记录风险轨迹与能力变化，形成可回放训练链。

### 讲稿提示
这不是“先写剧情再硬套训练”，而是“训练目标驱动剧情生成”。

### 可视化建议
- 协同环路图：State -> Director -> Script -> User -> State。

## 第9页 技术选型（为何这样选）
### 页面标题
技术选型：既要可靠可控，也要可持续迭代

### 页面要点
- 后端：FastAPI + Pydantic + SQLAlchemy + Alembic（契约稳定、事务可控、演进成本低）。
- 前端：React + TypeScript + Vite（训练页与报告页迭代快）。
- 模型能力：文本评估/剧情生成 + 生图 + 语音，采用可替换供应商策略。
- 架构原则：策略层解耦、Agent分工、主链路和异步媒体链路分离。

### 讲稿提示
选型核心不是“最潮”，而是“可解释、可维护、可落地”。

### 可视化建议
- 对比表：目标能力 / 选型组件 / 选择理由。

---

## 第10页 创新点与落地价值
### 页面标题
创新点总结：从“会生成”走向“会训练”

### 页面要点
- 创新1：KS状态驱动，不是单次打分。
- 创新2：双Agent协同（导演控节奏，剧本保连续）。
- 创新3：结果可解释（证据链、风险轨迹、能力曲线）。
- 创新4：可教学落地（课堂组织可控，个体训练可差异化）。

### 讲稿提示
一句话总结：我们做的是“会根据学生表现动态调整训练路径”的系统，而不是普通剧情对话系统。

### 可视化建议
- 四象限价值图：教学价值 / 工程价值 / 数据价值 / 推广价值。
